# 异步编程概念
Ref: example/async_function.js
* 等待
    * `async funciton`: 函数内部有异步操作；返回值是一个Promise对象。
    * `await`: 只能在 `async function` 内部使用的关键字。用来暂停代码执行，等待一个 Promise 解决（Resolve）并拿到最终结果。
    * `Promise`: 表示一个异步操作的最终完成（或失败）及其结果值的表示。
* 生成
    * `yield`: 只能在生成器函数内部使用的关键字。它的作用有两个：向函数外部吐出一个值，同时立刻暂停当前函数的执行。
    * `function*`: 同步数据流，一个可以暂停执行和恢复执行的特殊函数。调用它不会立刻执行内部代码，而是返回一个迭代器对象（Iterator）。
* `async function*`: 如果你想在一段充满异步操作的代码里，源源不断地向外吐出中间数据，就必须用它。

### JavaScript 函数形态终极对比


| 函数定义形态 | 能用 `await` 吗？ | 能用 `yield` 吗？ | 返回值是什么？ | 适用场景 |
| :--- | :---: | :---: | :--- | :--- |
| `function` | ❌ 否 | ❌ 否 | 普通值 | 普通同步计算 |
| `async function` |  能 | ❌ 否 | **Promise** 对象 | 单次异步操作（如调一次 API 拿结果） |
| `function*` | ❌ 否 |  能 | **Generator** (同步迭代器) | 同步数据流（如斐波那契数列生成器） |
| `async function*` |  能 |  能 | **AsyncGenerator** (异步迭代器) | **持续的异步数据流（如 AI 流式打字机、聊天主循环）** |
